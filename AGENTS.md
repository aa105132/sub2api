# 仓库协作说明

## 🗺️ 代码地图优先

- 开始任何分析、排查、编码前，先查看根目录 `CODEMAP.md`。
- 优先根据 `CODEMAP.md` 提供的入口文件、核心结构体、路由位置开始定位，减少重复全仓搜索。
- 如果本次要改的模块还没有写进 `CODEMAP.md`，允许先正常检索代码；但任务结束前必须把新定位到的模块补进地图。

## ✍️ 修改后同步地图

- 只要改了代码、配置、路由、测试或文档，就要同步更新 `CODEMAP.md`。
- 每次补充代码地图时，至少写清楚下面几项：
  - 文件路径
  - 入口函数 / 结构体 / 路由
  - 该文件作用
  - 本次改动点
  - 关联文件
- 地图内容以“方便下次快速定位”为目标，优先写稳定入口，不写大段无结论的描述。
- 不要删除他人已有地图条目；如需调整，优先增量修正。

## 📌 维护建议

- 路由文件重点记录：路径前缀、鉴权方式、落到哪个 handler。
- handler 文件重点记录：入口方法、参数来源、调用了哪个 service。
- service 文件重点记录：核心方法、上游接口、失败清理、副作用。
- 配置文件重点记录：新增配置项名、默认值、影响范围。
- 测试文件重点记录：覆盖的主流程和关键断言，方便回归。
